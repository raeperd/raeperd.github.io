# Immutability we can afford
- Immutable object is more expensive than mutable object. Because machines are fundamentally mutable.
- We can afford it now. Since our machines are much much powerful than before
- And it is actually reasonable costs. As developers time is expensive than machines, saving time by using immutable objects is not expensive at all.
- Modern languages like kotlin, swift, scala are prefer immutable object to mutable object
